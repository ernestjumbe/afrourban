# RECIPES

Use these compositions when building new pages.

## 1. Editorial Index Page

Use for stories, quizzes, and mixed content indexes.

Structure:

1. gold trim
2. sticky nav
3. short hero or page intro
4. category/filter rail
5. featured card or featured split
6. paginated grid of editorial cards

Rules:

- one item may get oversized feature treatment
- rest of the content sits in a disciplined grid
- keep search and filters visible but visually quiet

Example file:

- `examples/recipes/editorial-index-page.tsx`

## 2. Directory Page

Use for organizations or other searchable listings.

Structure:

1. gold trim
2. sticky nav
3. concise page intro
4. search field plus filters plus sort
5. listing cards
6. pagination

Rules:

- avoid dashboard aesthetics
- filters should feel editorial and curated
- cards should remain image or identity led

Example file:

- `examples/recipes/directory-page.tsx`

## 3. Auth Page

Use for login and registration.

Structure:

1. gold trim
2. sticky nav
3. centered or offset auth panel
4. mode toggle if multiple auth methods exist
5. form fields with concise helper text
6. strong primary CTA

Rules:

- keep the shell branded and calm
- inputs remain minimal
- validation should be clear, not noisy

Example file:

- `examples/recipes/auth-page.tsx`

## 4. Story Detail Page

Use for long-form editorial content.

Structure:

1. hero area with title and meta
2. cover image
3. article body column
4. optional sticky side rail for author/share/actions
5. related stories

Rules:

- serif headline is required
- body width must remain readable
- keep ornament around the content, not inside the prose column

Example file:

- `examples/recipes/story-detail-page.tsx`

## 5. Profile or Organization Detail

Use for identity-led detail pages.

Structure:

1. cover or banner
2. overlapping identity element such as avatar or logo
3. title and descriptive copy
4. action row
5. stats, related content, or supporting sections

Rules:

- let imagery lead
- keep actions compact and well-grouped

Example file:

- `examples/recipes/identity-detail-page.tsx`

## 6. Feature Landing Page

Use when combining multiple content types on one page.

Structure:

1. hero split or asymmetrical lead block
2. supporting story strip
3. upcoming events or secondary list
4. archive or newsletter side column

Rules:

- vary module sizes intentionally
- keep whitespace generous so the page still reads editorial

Example file:

- `examples/recipes/feature-landing-page.tsx`
